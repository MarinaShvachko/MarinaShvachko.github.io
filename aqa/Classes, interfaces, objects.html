Abstract class vs interface and a little about objects

Class

Is a template of an entity, which defines its initial state and behaviour. It's like a blueprint for objects. State is it's variables. Behaviour is it's methods.

In addition to constructors an initialization can be made with the help of initializer block that are performed before any constructors.

  
    public class Animal {

    public int speed;
    public int maxSpeed;

    {
        speed = 5;
    }
    }

Also there could be "static initialization blocks". Static variables and methods mean that all objects of a class will use this the same field \ method.

  
    public class Animal {

    private static int speed;
    private int maxSpeed;

    static {
        speed = 5;
    }
}

If you need to use static variables \ methods from another package, you can use "static" word in import block, so you will be able to call static members of a class without specifying a name of the class.

In order to forbid inheritance from a class, it is required to mark it with a modifier final. Methods marked as final cannot be overridden.

Object

Is an existing instance of an entity (class), which possesses certain state and behaviour. Static fields exist before an object of a class is created. Static methods can not access a non-static fields and methods.

Abstract class vs Interface

Common

we can not create an object of abstract class and interface can have methods without body (they are abstract and have to be implemented) and with body

Differences

Abstract class: we can extend only one class can have final and not final variables Interface: can implement a lot of interfaces all variables are final even if we don't write final word
